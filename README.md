# 🛍️ Product Recommendation Engine

A fully functional recommendation engine that suggests products to users using **Content-Based Filtering**, **Collaborative Filtering**, and a **Hybrid** approach that combines both. Built with Python, scikit-learn, and Streamlit.

![Python](https://img.shields.io/badge/python-3.9%2B-blue)
![License](https://img.shields.io/badge/license-MIT-green)

## ✨ Features

- **Content-Based Filtering** — TF-IDF vectorization of product metadata (category, brand, description) + cosine similarity to recommend items similar to what a user has liked.
- **Collaborative Filtering** — Two implementations:
  - *Item-based memory CF* using cosine similarity over the user-item rating matrix.
  - *Model-based CF* using Truncated SVD (matrix factorization) to predict ratings for unseen products.
- **Hybrid Recommender** — Weighted combination of content-based and collaborative scores, with automatic **cold-start handling** (falls back to popularity-based recommendations for brand-new users).
- **Evaluation suite** — Precision@K, Recall@K, and RMSE on a held-out test split.
- **Interactive Streamlit app** — Browse recommendations, explore similar products, and view dataset analytics in your browser.
- **CLI tool** — Get recommendations straight from the terminal.
- **Synthetic dataset generator** — Produces a realistic e-commerce catalog (150 products across 6 categories) and 6,000 user-product ratings from 300 users with believable preference patterns.

## 🚀 Quickstart

### 1. Clone and install dependencies

```bash
git clone https://github.com/<your-username>/product-recommendation-engine.git
cd product-recommendation-engine
pip install -r requirements.txt
```

### 2. Generate the dataset

```bash
python data/generate_data.py
```

This creates `data/products.csv`, `data/users.csv`, and `data/ratings.csv`.

### 3. Train the model

```bash
python train.py
```

This trains the hybrid recommender, evaluates it, and saves it to `models/hybrid_recommender.pkl`.

### 4. Get recommendations

**Via CLI:**
```bash
python recommend.py --user_id 5 --top_n 10
python recommend.py --user_id 5 --mode content       # content-based only
python recommend.py --user_id 5 --mode collaborative # collaborative only
python recommend.py --list_users                     # see sample user IDs
```

**Via interactive web app:**
```bash
streamlit run app.py
```
Then open the URL shown in your terminal (usually `http://localhost:8501`).

## 🧠 How It Works

### Content-Based Filtering
Each product's category, brand, name, and description are combined into a text "soup" and vectorized with **TF-IDF**. A user's profile is built as the weighted-average TF-IDF vector of products they rated highly, and we recommend the unseen products with the highest **cosine similarity** to that profile.

### Collaborative Filtering
Built from the user-item ratings matrix alone (no product metadata):
- **Item-based CF**: computes cosine similarity between products based on how users rated them, then predicts a user's rating for unseen items as a similarity-weighted average of their existing ratings.
- **SVD (matrix factorization)**: decomposes the mean-centered user-item matrix into latent factors using Truncated SVD, then reconstructs predicted ratings for every user-item pair.

### Hybrid
Normalizes and combines content-based similarity scores and collaborative-filtering predicted ratings using a configurable weighted sum (default: 40% content, 60% collaborative). For users with zero ratings (cold start), it falls back to a popularity score (`avg_rating × log(1 + num_ratings)`).

### Evaluation
- **Precision@K / Recall@K** — measured by holding out 20% of ratings, training on the rest, and checking how many of a user's held-out "liked" products (rating ≥ 4) appear in their top-K recommendations.
- **RMSE** — measures how accurately the SVD model predicts held-out ratings.

## 🔧 Using Your Own Data

Replace `data/products.csv` and `data/ratings.csv` with your own data using this schema:

**products.csv**: `product_id, product_name, category, brand, description, price`

**ratings.csv**: `user_id, product_id, rating` (rating scale 1–5)

Then re-run `python train.py`.
